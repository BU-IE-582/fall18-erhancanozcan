1) Introduction

Predicting the outcome of the soccer games in the Premier League is the purpose of this study. The data that we worked on is providing us information about the match results and bookmakers odds of the old games. By using this information, we tried to achieve the best predictions of the match result probabilities. While submitting probabilities during 8 rounds, we continued to adapt our models and methods.

2) Methodology

We started to work on the data by feature extraction and tried to add many features that are meaningful to us. At the end of this process, we got 313 features. Some of them were match results related features, others were odd related features. By using these features, we constructed models and selected parameters with the help of the cross-validation technique. To do that, the 2018-2019 season is separated to use as test data while old data is used as training data. Also, mean rps value is used as a performance measure. After constructing various models, we observed that the models are providing us approximately the same results and none of them is outperforming others. This conclusion led us to decrease the number of features and to find the most meaningful ones between them to prevent complexity. Therefore, we eliminated some of the features using random forests’ mean decrease Gini value and ended up with 100 features. Finally, we were able to construct models and interpret them with the knowledge of the previously constructed models. We learned from the trials of the models with 313 features, the decision tree was generally good to use because of its simplicity and good results. Thus, we decided to utilize decision tree firstly. Although the decision tree model was good, in some leaves we didn’t observe good rps values. Therefore we adapt our prediction method in such a way that in the case of our test data goes into the unsuccessful leaves, we alternate the model and use logistic regression.

matches_data_preprocessing <- function(data){
  
  temp = copy(data)
  temp = unique(temp,by="matchId") 
  setnames(temp,c("home","away","score","date"),c("Home","Away","Score","Match_Date"))
  temp[,Home:=tolower(Home)]
  temp[,Away:=tolower(Away)]
  temp[,Match_DateTime:=as.POSIXct(Match_Date,tz="UTC",origin = as.POSIXct("1970-01-01",tz="UTC"))]
  temp[,Match_Hour := format(strptime(Match_DateTime,"%Y-%m-%d %H:%M:%OS"),'%H')]
  temp[,Match_Hour := as.numeric(Match_Hour)]
  temp[,Match_Date := as.Date(Match_DateTime,format="%Y-%m-%d")]
  
  temp[,AWA_FLAG:=0]
  temp[(Score %like% "AWA."),`:=`(Score=gsub("AWA.","",Score),AWA_FLAG=1)]
  temp[,POSTP_FLAG:=0]
  temp[(Score == "POSTP."),`:=`(Score=gsub("POSTP.","",Score),POSTP_FLAG=1)]
  temp[,CAN_FLAG:=0]
  temp[(Score == "CAN."),`:=`(Score=gsub("CAN.","",Score),CAN_FLAG=1)]
  latestDateTimeofKnownScore = max(temp[Score!=""]$Match_DateTime)
  POSTP_toberemoved = temp[which(temp$POSTP_FLAG==1 & temp$Match_DateTime <= latestDateTimeofKnownScore)]$matchId
  if(length(POSTP_toberemoved)!=0){
    cat("Following postponed matches are REMOVED during data_preprocessing:\n")
    print(data[matchId%in%POSTP_toberemoved,])
    temp=temp[!matchId%in%POSTP_toberemoved,]
  }
  POSTP_tobekept = temp[which(temp$POSTP_FLAG==1 & temp$Match_DateTime > latestDateTimeofKnownScore)]$matchId
  if(length(POSTP_tobekept)!=0){
    cat("Following postponed matches are KEPT during data_preprocessing:\n")
    print(data[matchId%in%POSTP_tobekept,])
  }
  
  temp[,c("Home_Score","Away_Score") := lapply(tstrsplit(Score,":",fixed=T),as.integer)]
  temp[,`:=`(Match_Result = ifelse(Home_Score == Away_Score, "Tie" , ifelse(Home_Score > Away_Score, 'Home' , 'Away'))
             ,Total_Score = Home_Score + Away_Score)]
  temp[,`:=`(Result_Home = ifelse(Match_Result=="Home",1,0)
             ,Result_Tie = ifelse(Match_Result=="Tie",1,0)
             ,Result_Away = ifelse(Match_Result=="Away",1,0))]
  
  temp[,c('Score','Match_DateTime','AWA_FLAG','POSTP_FLAG','CAN_FLAG'):=NULL]
  gc()
  return(temp)
}

#' Data Preprocessing for Odd Details Data
#'
#' Makes preprocessing on raw odd details data so that it can be used by other functions.
#' only 1x2 bets are considered, if you are interested in other types of odds manipulate ''which_bets''
#' @param data A data.table containing raw odd details data
#' @param remove_bookmaker A character vector containing the bookmakers to be removed
#' @export
#' @examples

details_data_preprocessing <- function(data,matches,which_bets=c('1x2'),remove_bookmaker=c('BetfairExchange','PaddyPower'),removeOlderThan=30){
  # data manipulation for historical odd data
  details = copy(data)
  
  #remove duplicate entries
  details = unique(details)
  
  details = details[betType %in% which_bets]
  details[,totalhandicap:=NULL]
  
  details = merge(details,matches[,list(matchId,Match_Date)],by="matchId",all.x=T)
  setnames(details,"date","OddChangeDateTime")
  details[,OddChangeDateTime:=as.POSIXct(OddChangeDateTime,tz="UTC",origin = as.POSIXct("1970-01-01",tz="UTC"))]
  details = details[difftime(Match_Date,OddChangeDateTime, units = "days") <= removeOlderThan] #remove odds seen earlier than 10 days from the match date
  details[, odd := as.numeric(odd)]
  
  details[,bookmaker:=gsub(" |-","",bookmaker)]
  if(!is.null(remove_bookmaker)){
    details = details[!(bookmaker %in% remove_bookmaker)]
  }
  
  gc()
  return(details)
}

  details<-as.data.table(left_join(details,feature_odd_details_tmp,by=c("matchId"="matchId")))
  return(features = details)

train_glmnet <- function(train_features, test_features,not_included_feature_indices="Match_Result", alpha=1,nlambda=50, tune_lambda=TRUE,nofReplications=1,nFolds=10,trace=T){
  
  set.seed(1)
  
  # glmnet works with complete data
  glm_features=train_features[complete.cases(train_features)]
  train_class=glm_features$Match_Result
  glm_train_data=glm_features[,-not_included_feature_indices,with=F]
  #test featurelar?? haz??rla
  glm_test_data=test_features[,-not_included_feature_indices,with=F]
  if(tune_lambda){
    # to set lambda parameter, cross-validation will be performed and lambda is selected based on RPS performance
    #set.seed(1)
    #cvindices=generateCVRuns(train_class,nofReplications,nFolds,stratified=TRUE)
    
    # first get lambda sequence for all data
    glmnet_alldata = glmnet(as.matrix(glm_train_data), as.factor(train_class), family="multinomial", alpha = alpha, nlambda=nlambda,standardize = TRUE)
    lambda_sequence = glmnet_alldata$lambda
    
    cvresult=vector('list',nofReplications*nFolds)
    iter=1
    
    
    for(i in 1:nofReplications) {
      thisReplication=indices[[i]]
      for(j in 1:nFolds){
        if(trace){
          cat(sprintf('Iteration %d: Fold %d of Replication %d\n',iter,j,i))
        }
        testindices=(thisReplication[[j]])
        
        cvtrain=glm_train_data[-testindices]    
        cvtrainclass=train_class[-testindices]   
        cvtest=glm_train_data[testindices]
        cvtestclass=train_class[testindices] 
        
        inner_cv_glmnet_fit = glmnet(as.matrix(cvtrain),as.factor(cvtrainclass),family="multinomial", alpha = alpha,lambda=lambda_sequence,standardize = TRUE)
        valid_pred = predict(inner_cv_glmnet_fit, as.matrix(cvtest), s = lambda_sequence, type = "response")
        
        #check order of predictions
        order_of_class=attr(valid_pred,'dimnames')[[2]]
        new_order=c(which(order_of_class=='Home'),which(order_of_class=='Tie'),which(order_of_class=='Away'))
        foldresult=rbindlist(lapply(c(1:length(lambda_sequence)),function(x) { data.table(repl=i,fold=j,lambda=lambda_sequence[x],valid_pred[,new_order,x],result=cvtestclass)}))
        cvresult[[iter]]=foldresult
        iter=iter+1
      }
    }
    
    cvresult=rbindlist(cvresult)
    
    # creating actual targets for rps calculations
    cvresult[,pred_id:=1:.N]
    outcome_for_rps=data.table::dcast(cvresult,pred_id~result,value.var='pred_id')
    outcome_for_rps[,pred_id:=NULL]
    outcome_for_rps[is.na(outcome_for_rps)]=0
    outcome_for_rps[outcome_for_rps>0]=1
    setcolorder(outcome_for_rps,c('Home','Tie','Away'))
    
    outcome_for_rps[,Tie:=ifelse(Home==1,1,Tie)]
    outcome_for_rps[,Away:=ifelse(Tie==1,1,Away)]
    
    #prediction probabilities
    setnames(cvresult,old=c("Home","Tie","Away"),new=c("Home_prob","Tie_prob","Away_prob"))
    cvresult<-cbind(cvresult,outcome_for_rps)
    cvresult_probs<-apply(cvresult[,c(4,5,6)],1,cumsum)
    cvresult_probs<-as.data.table(t(cvresult_probs))
    #calculation of RPS
    rps<-((cvresult_probs$Home_prob-outcome_for_rps$Home)^2 +
            (cvresult_probs$Away_prob-outcome_for_rps$Away)^2 +
            (cvresult_probs$Tie_prob-outcome_for_rps$Tie)^2)/2
    
    rps<-as.data.table(rps)
    cvresult<-cbind(cvresult,rps)
    
    
    
    
   
    # find best lambdas as in glmnet based on RPS
    overall_results_summary=cvresult[,list(meanRPS=mean(rps),sdRPS=sd(rps)),list(lambda)]
    overall_results_summary[,RPS_mean_lb := meanRPS - sdRPS]
    overall_results_summary[,RPS_mean_ub := meanRPS + sdRPS]
    
    cv_lambda_min=overall_results_summary[which.min(meanRPS)]$lambda
    
    semin=overall_results_summary[lambda==cv_lambda_min]$RPS_mean_ub
    cv_lambda.1se=max(overall_results_summary[meanRPS<semin]$lambda)
    
    cvResultsSummary = list(lambda.min =cv_lambda_min, lambda.1se = cv_lambda.1se,
                            meanRPS_min=overall_results_summary[lambda==cv_lambda_min]$meanRPS,
                            meanRPS_1se=overall_results_summary[lambda==cv_lambda.1se]$meanRPS)
    
  }
  
  # fit final glmnet model with the lambda with minimum error
  final_glmnet_fit = glmnet(as.matrix(glm_train_data),as.factor(train_class),family="multinomial", alpha = alpha,lambda=cvResultsSummary$lambda.min,standardize = FALSE)
  # obtain predictions
  predicted_probabilities=predict(final_glmnet_fit, as.matrix(glm_test_data), type = "response")
  
  #check order of predictions
  order_of_class=attr(predicted_probabilities,'dimnames')[[2]]
  new_order=c(which(order_of_class=='Home'),which(order_of_class=='Tie'),which(order_of_class=='Away'))
  
  final_result=data.table(test_features[,list(matchId,Match_Result)],predicted_probabilities[,new_order,1])
  
  return(list(predictions=final_result,cv_stats=cvResultsSummary))
}

4) Future Work

First of all, additional match-related features can be further improved. We obtained the cumulative number of goals conceded by the away team in this season in all matches, the cumulative number of goals scored by the home team in this season in all matches and so on. These features can also be divided into 2 categories such that the statistics of the home team can be obtained from the previous home matches and from the previous away matches seperately. The same procedure can be applied to the away team, as well. This may improve the performance of the models.

Secondly, in the model construction part, the leaves of the decision tree are examined. As mentioned above, the leaves having 757 matches and 322 matches are seperated, and for the matches falling to the other leaves, logistic regression is applied. However, it can be thought that these imbalances in mean rps of the leaves can be due to class imbalance problem because there are very few number of matches that ended as ties compared to home and away wins. However, in the seperated 2 leaves, the majority is home and away wins. Thus, seperating those matches can decrease the class imbalance problem. As an additional step, while predicting the matches that fall to the remaining leaves with logistic regression, the model can be trained by the remaining 1158 matches instead of all 2237 matches. This may improve the performance since class imbalance problem is decrease to some extent.
